# Question Generation Workflow

## Overview

Questions are generated by running the same source passage through 4 LLMs independently, then manually reviewing and deduplicating candidates before adding to the master dataset. This multi-model approach produces diversity in question framing, distractor choice, and concept coverage, and enables a post-hoc self-generation bias analysis during evaluation.

## Generation Models

| Model | Interface | `generated_by` value |
|---|---|---|
| Claude Sonnet | claude.ai | `Claude Sonnet` |
| ChatGPT (GPT-4o / GPT-4.1) | chatgpt.com | `ChatGPT` |
| Perplexity | perplexity.ai | `Perplexity` |
| DeepSeek | chat.deepseek.com | `DeepSeek` |

## Step-by-Step Process

### 1. Select a source passage
- Open the relevant IPCC chapter PDF
- Choose a section that maps cleanly to the target category (verify content matches category — not just chapter number)
- For equation-heavy sections: take a screenshot and upload as image rather than copy-pasting LaTeX
- For mostly text sections with inline equations: copy-paste and write equations in plain notation (e.g. `lambda = delta_F / delta_T`)
- Avoid pure derivation sections — they don't produce good MCQ source material

### 2. Generate prompt files
Run the prompt generation script from the `climate_benchmarking/` directory:

```bash
python generate_prompts.py \
  --section wg1_7.4.2.1 \
  --category "Physical Climate Science Fundamentals" \
  --source "IPCC AR6 WG1 Chapter 7, Section 7.4.2.1" \
  --passage passage.txt
```

This creates `prompts/tests/{section}/{model}/prompt.md` for all 4 models with placeholders filled. Save your passage text to `passage.txt` first.

### 3. Run each model in a fresh session
- Open a new chat session per model (do not reuse sessions across batches)
- Paste the contents of each model's `prompt.md` directly into the chat
- For image-based passages: paste the prompt text first, then upload the screenshot
- Copy the JSON response into `prompts/tests/{section}/{model}/response.json`

### 4. Review and curate
Review all responses together:
- Verify every correct answer against the source passage
- Cut questions where the correct answer is ambiguous or requires outside knowledge
- Cut duplicate concepts across models — keep the version with tighter distractors
- Check difficulty distribution: each batch should be ~2 easy, 2 medium, 1 hard across kept questions (not per model)
- Fix any source field errors (wrong chapter number, missing section depth)
- Target acceptance rate: ~60% (standard), ~50% (Category 2 data interpretation)

### 5. Add to master dataset
Append accepted questions to `data/questions.json` with all fields populated including `generated_by`.

## Key Design Decisions (for case study)

**Multi-model generation:** Using 4 models produces diversity in framing and distractor choice that a single model cannot. It also enables self-generation bias analysis: do models score higher on questions they generated? Tracking `generated_by` for every question makes this analysis possible with no extra work.

**Fresh sessions per batch:** Each batch is run in a new chat session to prevent in-context bias from prior question generations influencing distractor or concept choices.

**Manual review over automation:** All questions are human-reviewed before entering the master dataset. Automated quality checks can catch format errors but cannot reliably verify that a distractor is genuinely unambiguous. Human review is the quality gate.

**~60% target acceptance rate:** Roughly 2 of every 5 generated candidates are cut — either for concept overlap, weak distractors, ambiguous correct answers, or wrong difficulty calibration. This is intentional; generating more candidates than needed and filtering is cheaper than trying to get every question right on the first pass.
